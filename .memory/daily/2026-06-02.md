**Decisions Made:**
- Для runtime restart выбран structural fix: app-level coordinator больше не делает отдельные STOP/START foreground-service intents, а отправляет single service action `ACTION_RESTART_RUNTIME_CONFIG`.
- `OzeroVpnService` должен выполнять restart внутри уже запущенного service: stop без `stopSelf`, затем возврат foreground и `startVpn()`.

- `performShutdown(callStopSelf=false)` всё равно снимает foreground notification, поэтому service-side restart обязан вернуть foreground перед внутренним `startVpn()`.
- Runtime restart baseline race возникает в окне между enqueue start и фактическим входом `StartSequenceCoordinator` в starting state.
- Watchdog path может обойти stale-signal guard `TunnelController.onEngineDied`, если сам включает killswitch или вызывает `stopVpnRequest()` до проверки активного движка.
