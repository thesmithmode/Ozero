**Key Exchanges:**
- Ассистент несколько раз амеднил и пушил коммит в `dev`, ориентируясь на ручную simulation coverage XML, после чего CI снова запускался и падал на `Tests — app`.

**Lessons Learned:**
- Нельзя пушить новый CI-run, если fix не доказан по свежему artifact и реальной Gradle/Jacoco mask-логике.
- Ручная simulation JaCoCo XML может не совпасть с Gradle exclude matching; это нельзя считать достаточным доказательством.
- Широкие excludes вроде `**/ui/**` могут ухудшить ratio, если убирают покрытый код, а miss-heavy classes остаются.
- Coverage job сейчас смешивает compile, unit tests, N>0 discovery, Jacoco report, verification и artifact upload, что усложняет отладку.

**Action Items:**
- Перед следующим push доказать, что `jacocoTestCoverageVerification` станет ≥95% по фактическим class names/masks.
- Рассмотреть последующее разбиение `Tests — app` на диагностические jobs после стабилизации coverage policy.

**Decisions Made:**
- Сначала чинить coverage, не дробить CI и не пушить проверочные runs.
- Цель фикса: честные `LINE >= 0.95` и `BRANCH >= 0.95` для `:app:jacocoTestCoverageVerification`.

- Исправлять либо focused unit tests для production-important кода, либо узкими excludes для Android glue/UI/generated/native boundary с sentinel в `JacocoExcludesContractTest`.
- Не снижать threshold, не отключать verification, не добавлять широкие exclude-маски.
- Разбиение `Tests — app` делать только после зелёного coverage, чтобы не менять две переменные одновременно.

**Lessons Learned:**
- CI push нельзя использовать как проверку гипотезы; перед запуском нужен доказанный fix по свежему report и реальной Gradle mask-логике.
- Фактическое попадание compiled classes под JaCoCo excludes важнее визуальной оценки масок по исходникам.
- Sharding тестов по фильтрам опасен ложной зелёнкой, если на каждом shard нет отдельного N>0 gate.

**Decisions Made:**
- Из JaCoCo excludes нужно оставлять только generated/DI/Android shell/native/external SDK wrappers.

- Не добавлять дублирующие тесты там, где содержательное покрытие уже есть; вместо этого усилить contract-тест на coverage boundary.
- Добавить прямые unit-тесты для `RuntimeFailureRouter`, так как у него есть небольшая branch/state логика без явного покрытия.

**Lessons Learned:**
- `JacocoExcludesContractTest` должен защищать не только от широких масок, но и от точечных исключений тестируемых классов.
- Пустой bootstrap seed не является новой поломкой `FirstRunBootstrap`, потому что тот уже был no-op в релизе; изменение относится именно к bundled asset.
- В Ozero локальные Gradle/test/lint проверки запрещены, валидация только через CI.
