### Session (16:02+) — URnetwork UI стабилизация

**Context:** Пользователь видит нестабильный экран параметров URnetwork: секции пропадают/появляются при подключении, баланс не показывается когда не подключён.

**Проблемы найдены:**
1. `BalanceCard` рендерился только в `Ready` state — не виден при NotConnected
2. `ProvideSection` скрыта если `tunnelState != URNETWORK` — исчезает при смене движка
3. Race в `LocVM.init`: два `launch` без синхронизации → NotConnected-флаш при открытии экрана
4. `FREE_TIER_CAP_BYTES = 34 GiB` обрезал реальный баланс включая бонусные GB
5. **RelayCoordinator hardcode bug**: строки 67+81 всегда вызывали `bridge.setProvidePaused(false)` — игнорировали configStore. При подключении URnetwork-движка relay перезаписывал результат `EngineUrnetwork.start()`.

**Fixes (commit e0d53ca4):**
- `UrnetworkEngineSettingsViewModel`: добавлен `providePaused: StateFlow` из configStore
- `UrnetworkEngineSettingsScreen`: BalanceCard в NotConnected branch, ProvideSection без `showProvide` gate
- `UrnetworkLocationsViewModel`: `bootstrapJob.join()` перед active/inactive веткой — устраняет race
- `UrnetworkBalanceCard`: убран `FREE_TIER_CAP_BYTES`, показывается реальный баланс
- `UrnetworkRelayCoordinator`: для URNETWORK-движка — не перезаписывать (engine уже применил configStore); для relay — читать `configStore.provideEnabled()` вместо хардкода false

**Lessons Learned:**
- `UrnetworkRelayCoordinator` не имел доступа к configStore для provideEnabled — баг был с первого коммита `194d7701`, стал заметен только когда UI дал возможность менять настройки из NotConnected состояния
- bootstrapJob.join() pattern: inject-before-decide — гарантирует что init завершён до принятия решения о NotConnected
- cap 34 GiB скрывал реальный баланс: убран по решению пользователя, бэкенд-баг не client-fixable
