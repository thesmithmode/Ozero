### Session (11:35)

**Context:** Работа над Ozero: исправление P2 Singbox runtime fingerprint bug и массовое добивание test/coverage gaps до 95%.

**Key Exchanges:**

- Ассистент изменил тактику: меньше широких Gradle-прогонов, больше пакетного написания тестов по Jacoco XML и targeted запусков.
- Пользователь запросил промежуточные итоги после примерно часа работы, оценку темпа и меры ускорения.

**Decisions Made:**
- Для Singbox runtime fingerprint выбран root fix: при cache miss выбранного профиля синхронно читать `profileDao.getById()`, не использовать stale `BEAN_KEY` как fallback.
- Для ускорения coverage recovery принято писать по 5-10 тестов за заход в одном модуле и запускать только конкретные test class через `--tests`; полный module test запускать только после стабилизации пачки.
- Приоритет coverage изменён: сначала чинить текущие failing app tests, затем добивать near-threshold modules (`common-dns`, `singbox-config`, `singbox-fmt`, `singbox-subscription`, `engines-core`), затем крупные модули по high-value classes из Jacoco.
- Для больших Android-service классов решено избегать хрупких source-sentinel tests, если можно тестировать чистую логику или existing coordinator harness.

**Lessons Learned:**
- `Build.MANUFACTURER`/`Build.MODEL` в Robolectric/runtime могут быть `null` несмотря на non-null сигнатуру; `RomCompat` должен использовать `.orEmpty()`.
- Kotlin не поддерживает `\f` в обычной строке; для таких ожиданий нужен raw literal или другой безопасный формат.
- Некоторые existing tests могут не попадать в Jacoco coverage из-за build/report mapping issue, поэтому для быстрого роста покрытия лучше выбирать классы, где Jacoco точно считает execution.
- Brittle sentinel tests на точную строку с indent ломаются от форматирования; лучше использовать структурный anchor внутри нужной функции.
- Существующие файлы с mojibake мешают patch-контексту; для новых тестов иногда быстрее создать отдельный ASCII-only файл.

**Action Items:**

**Decisions Made:**

- Base64 decoder в `singbox-fmt` permissive: некоторые invalid-looking строки могут декодироваться в мусорную UTF-8 строку через fallback, поэтому invalid assertion нельзя строить на произвольных символах.
