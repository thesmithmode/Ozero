**Context:** Ozero Android VPN — фиксы GROUP A (ByeDPI) + GROUP B (URnetwork), начало GROUP C

**Key Exchanges:**
- Root cause A1/A3: `buildTunBuilder()` не передавал `excludeSelf=true` для SOCKS-пути → трафик Ozero уходил в свой же VPN → бесконечная петля через hev→ciadpi→VPN

- B3 root cause: MainViewModel использовал статический ключ `"URNETWORK:0"` → при смене страны IP не переопрашивался; решение — polling-корутина каждые 5с при активном URnetwork

**Decisions Made:**
- `IP_INFO_WARMUP_MS`: 8000 → 3000, ретраи 4→3, задержка 1000→1500
- B3 реализован через `while(true) + delay(URNETWORK_LOCATION_POLL_MS)` в `viewModelScope` вместо key-invalidation
- `while(true)` в viewModelScope при URnetwork → сломал `advanceUntilIdle()` в тестах → заменил на `advanceTimeBy()` + `runCurrent()`
- Sentinel `IP_INFO_WARMUP_MS не меньше 8 секунд` обновлён до 3000

**Lessons Learned:**
- `while(true) + delay()` в init/viewModelScope → `advanceUntilIdle()` вечный цикл в тестах; использовать `advanceTimeBy()` + `runCurrent()`
- Kotlin `"" ?: fallback` = `""`, не `fallback` — пустая строка не null; защита через `takeIf { isNotBlank() } ?: fallback`
- При refactoring conditional параметров (excludeSelf default=false) проверять ВСЕ call-sites — один пропущенный = регрессия
