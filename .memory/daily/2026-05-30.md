- В review найдены риски: missing import `IpInfo` в `MainViewModel`, stale `activeSocksPort` в `SingboxEngine`, serial FPTN auth fallback `15s * N`, mojibake в изменённых строках.

**Decisions Made:**
- Для review использовать `origin/main..origin/dev`, а не локальный `main`, потому что локальный `main` устарел.
- `SingboxEngine`: `activeSocksPort` должен означать только реально активный SOCKS probe, а не просто выделенный порт; публиковать порт только после успешного старта runtime/proxy.
- `FptnEngine`: убрать serial auth ladder из критического startup path; startup должен выбирать один сервер и делать один bounded auth attempt.
- Для FPTN безопаснее восстановить семантику `v0.2.0`: explicit selected server, иначе первый сервер из токена. Upstream-like fastest selection считать отдельной фичей, не текущим фиксoм.
- Full fallback для FPTN не должен выполняться внутри обычного startup под mutex оркестратора; если нужен, то отдельный cancellable recovery/diagnostic path.

**Lessons Learned:**
- FPTN upstream не логинится во все серверы подряд: сначала выбирает один сервер через availability logic, затем делает login только в него.
- Serial fallback по всем FPTN servers не подтверждён ни upstream, ни стабильным `v0.2.0`; он может блокировать `ChainOrchestrator` на `15s * N`.
- Mojibake может маскировать ktlint/compile проблемы после патчей через PowerShell или неверную кодировку; такие строки надо отдельно искать и чистить.
