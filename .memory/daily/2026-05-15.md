**Context:** Дебаг и фикс MTG генерации ключа (`engine-telegram`), затем ревью изменений агента.

**Key Exchanges:**
- Первоначально агент добавил только логирование — без реального диагноза. Юзер жёстко пресёк: "ты нихуя не починил".
- После `/gsd:debug` — найден реальный корень: `extractNativeLibs=false` (AGP default с 3.6) → `libmtg.so` не распаковывается при установке → `ProcessBuilder` падает с `binary.exists()=false` → UI молчит тихо.
- Фикс: `jniLibs.useLegacyPackaging=true` в `ozero.android.application.gradle.kts`.

**Decisions Made:**
- `useLegacyPackaging=true` в convention plugin (одно место для всех модулей) — правильный уровень фикса.
- Тест `MtgExtractNativeLibsSentinelTest` читает реальный buildscript с диска — нет мок-обмана.

**Lessons Learned:**
- `android:extractNativeLibs=false` — AGP default с 3.6. Любой Go-бинарь (`.so` как executable) в `jniLibs/` не распакуется без явного `useLegacyPackaging=true`. Паттерн применим и к будущим prebuilt-бинарям (`engine-telegram`, и другим subprocess-proxy).
- Shell-скрипты в тестах (`#!/bin/sh`) не запустятся на Windows — приемлемо, тесты Kotlin/Android только в CI (Linux).
- `locateRepoRoot()` в тесте обходит 5 уровней вверх — немного хрупко.
