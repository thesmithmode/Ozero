### 1. `EngineUrnetwork.stop()` не вызывает `sdkBridge.stop()`

**Файл:** `engine-urnetwork/src/main/java/.../EngineUrnetwork.kt`

```kotlin
override suspend fun stop() {
    // sdkBridge.stop() НЕ вызывается
    _state.value = EngineState.Stopped
}
```

**Риск:** URnetwork SDK (`Sdk.newNetworkSpaceManager()`) остаётся запущенным в памяти после вызова `stop()`. При повторном `start()` — дублирующий SDK instance. Утечка goroutines (Go runtime внутри AAR).

### 2. `ByeDpiEngineSettingsRepository` — `engineArgs` без валидации

**Файл:** `engine-byedpi/src/main/java/.../ByeDpiEngineSettingsRepository.kt`

`engineArgs` — строка из DataStore (введена пользователем через UI) → напрямую передаётся в `argv` парсер ByeDPI native:

```kotlin
// В ByeDpiProxy.kt:
val args = settingsRepository.getArgs() // raw string
val argv = listOf("byedpi") + args.split(" ") // split и в native
byeDpiStart(argv.toTypedArray())
```

**Риск:** Пользователь вводит аргументы → они попадают в `getopt_long` C-кода. Это не shell exec (нет `Runtime.exec(string)`) → классического command injection нет. Но неожиданные флаги могут:
- включить режимы с непредсказуемым поведением (`--disorder`, UDP trap)
- передать невалидный fd → SIGSEGV в native

**Реальная угроза:** Низкая (нет shell). Но sanitize всё равно нужен — минимум: strip null bytes, limit length, validate known flags.

---

private val mirrors = listOf(
