- Параллельно юзер описал баг: график скорости на главном экране деформирует исторические данные ("букву M колбасит") при сдвиге окна

**Decisions Made:**
- CI fix: `fetchSubscriptionBalance` переносится обратно из `UrnetworkApiHelper` в `RealUrnetworkSdkBridge` — тесты ожидают inline реализацию с `running.get()` gate и substring assertions на body
- `openLocationsViewController` тоже нуждается в `running.get()` gate (был только `deviceRef` check)
- Chart fix: root cause — `bucketize` группирует по относительной позиции в окне, не по timestamp → при сдвиге окна состав bucket'ов меняется → исторические средние "плывут". Решение: time-aligned buckets, anchored к wall-clock grid (кратные `bucketSec` от эпохи)
- Новый тип `SpeedSample` вместо голых `Pair`, функция `bucketizeTimeAligned` вместо старой `bucketize`

**Lessons Learned:**
- Bucket'изация по позиции в sliding window (index-based) = баг по определению: сдвиг окна меняет mapping точек → исторические данные "плывут". Всегда anchor к абсолютному времени
- Тесты contract-стиля с substring assertions на body функции хрупки при рефакторинге (перенос в helper класс ломает тесты)
