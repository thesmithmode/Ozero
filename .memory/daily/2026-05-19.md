### v0.1.5 — WARP/switch speedup + ByeDPI native diag (autonomous session)

Жалобы юзера: ByeDPI не даёт YouTube (хотя оригинальный ByeDPI APK даёт); WARP "медленно работает", "нестабилен", "ломается при передаче трафика"; "очень долгое переключение модулей". Юзер ушёл, дал автономию до релиза.

Лог `ozero.log` 2026-05-19 (34296 строк) — анализ через 3 параллельных general-purpose агента + advisor консультация (2 раза).

**Что найдено:**
- ByeDPI: native `main()` внутри upstream byedpi submodule **иногда висит** при cold-start. Лог: первый cold-start hung 5+ сек без crash (line 12-58); через 19s же успешен за 14ms (line 5642-5649). Race condition в byedpi C-code. НЕ исправлен в этой сессии без device-side debug.
- WARP: `awgTurnOn` возвращает `handle=0` (валидный — Go bridge даёт 0 первым slot'ом из tunnelHandles map; проверено по `.claude/Контекст/amnezia-client/.../api.go:123-128`). Но `awaitReady` **никогда не получает handshake confirmation** ни в одной WARP-сессии (10s timeout каждый раз). Старая логика: timeout → `Connected(WARP, port=0)` ЛОЖНО. Юзер видит "Connected" но 0 b/s.
- Switching: `SWITCHING_TIMEOUT_MS=12s` + WARP_READY=10s = долгая визуальная пауза при переключении.
- PORTAL_WG (`.claude/Контекст/PORTAL_WG_v1.4.3`) тоже использует AWG params (Jc/H1) → гипотеза "DEFAULT_WARP без обфускации" advisor'ом отвергнута (риск регрессии для пользователей в restrictive ISPs). DEFAULT_WARP preset не трогать.

**Применено:**
- `EngineWarp.WARP_READY_TIMEOUT_MS` 10s → 5s
- `TunnelController.SWITCHING_TIMEOUT_MS` 12s → 6s
- `StartSequenceCoordinator.awaitEngineReady` теперь Boolean; при Timeout → `engineWatchdog.handleEngineFailure` + `chainOrchestrator.stop` + return. False-Connected больше не наступает.
- `engine-byedpi/.../native-lib.c`: `__android_log_print` checkpoints вокруг `main(argc, argv)` + `jniStopProxy`/`jniForceClose` (tag `ByeDpiNative`). Следующая сессия увидит ГДЕ висит main().
- `EngineWarp.companion private → internal` для sentinel test.
- Sentinel: `EngineWarpTimeoutSentinelTest`, sentinel в `TunnelControllerTest`.

**Trade-offs (advisor):**
- Manual=WARP при handshake fail → disconnect вместо false-Connected. Acceptable — честно лучше тихого вранья.
- `WarpHandshakeUapi.check` soTimeout=50ms — теор. риск false-neg при медленном UAPI. Низкий риск (UAPI обычно <5ms). Не правил.
