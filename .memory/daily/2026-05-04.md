### Session (15:22)

**Context:** Фикс красного CI и release после hotfix OkHttp 4.12.0→5.3.0; выпуск v0.0.2

**Key Exchanges:**
- CI красный после hotfix: OkHttp 5.x требует Kotlin 2.2.0+, у проекта 2.0.20 → `compileReleaseKotlin FAILED`
- Первый откат трогал только `libs.versions.toml` → CI всё равно красный: `force("okhttp:5.3.0")` в `build.gradle.kts` имеет приоритет над version catalog
- Release fail: `StubUrnetworkSdkBridge` обнаружен в dex, release.yml запрещает stub-классы в APK → удалён из `main` (DI уже использует `RealUrnetworkSdkBridge`)

**Decisions Made:**
- Тег `v0.0.2` перебит дважды: после OkHttp fix и после stub fix
- `StubUrnetworkSdkBridge.kt` удалён из `main` source set полностью

**Lessons Learned:**
- `force()` в `build.gradle.kts` переопределяет `libs.versions.toml` → при откате dependency смотреть ОБА места
- Когда CI красный несколько раз подряд — читать ВСЕ ошибки run'а сразу, не по одной; первый симптом ≠ единственный корень
- Release check на forbidden stub classes — отдельный gate поверх зелёного CI; stub в main = release fail даже если DI не использует

**Action Items:**
- Дождаться завершения CI dev + Release v0.0.2 (watcher активен)
- После зелёного release — подтвердить APK собрался и доступен
