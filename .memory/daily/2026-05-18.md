### Session (10:41)

**Context:** Починка упавшего релиза v0.1.1 — sentinel в release.yml требовал libmtg.so для 3 ABI, APK содержит только arm64-v8a.

**Key Exchanges:**
- Release CI упал на sentinel-проверке `libmtg.so` для `armeabi-v7a` — ABI отсутствует в APK после коммита 5d35fc4b (abiFilters = arm64-v8a only)
- Все остальные sentinels (libhev, libbyedpi, libam) уже были arm64-v8a-only — только libmtg выбивался

**Decisions Made:**
- libmtg sentinel приведён к arm64-v8a-only (аналогично остальным native libs)
- CLAUDE.md инвариант про abiFilters обновлён — было "arm64-v8a, armeabi-v7a, x86_64" (устаревшее), стало "только arm64-v8a"

**Lessons Learned:**
- При добавлении нового native binary (libmtg) sentinel был скопирован с multi-ABI шаблона, не адаптирован под текущий abiFilters. Новые sentinels проверять на соответствие abiFilters.
- CLAUDE.md инварианты могут рассинхронизироваться с кодом — при изменении build config проверять документацию.

**Action Items:**
