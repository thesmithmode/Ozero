Lessons Learned:
- В coroutine unit tests недостаточно отменить только job ref: для infinite loop нужен cancel, drain scheduler и закрытие scope.
- `StandardTestDispatcher()` вне `runTest` может создать другой scheduler, из-за чего StateFlow/VM tests становятся flaky или stale.
- JVM unit tests не должны создавать platform-backed `ImageBitmap(1, 1)` для простой проверки delegation; нужен mock/stub.
- После изменения `profilesFlow` перед действием вроде `onPing` нужно дождаться collection/idle, иначе тест проверяет stale snapshot.
