**Key Exchanges:**
- Codex-агент добавил JetBrains MCP в `~/.codex/config.toml`, записав Windows-пути в double-quoted TOML строку → бэкслеши не экранированы → parse error при старте

**Decisions Made:**
- Исправление: double-quoted строку с Windows-путями заменить на TOML literal (single-quoted `'...'`) — в них бэкслеши не нужно экранировать

**Lessons Learned:**
- TOML: в double-quoted строках `\` = escape-символ → Windows-пути с `\` требуют `\\`. Решение: использовать single-quoted literals `'C:\path\...'` — безопасны для любых Windows-путей
- Codex-агент при редактировании TOML не учитывает тип кавычек → потенциальный источник broken configs при любом автоматическом изменении

**Action Items:**
