### Session (19:34)



**Context:** Ozero VPN — расследование silent crash на Android 15 (Nubia NX729J) при запуске/click VPN, multi-agent team debug session.

**Key Exchanges:**
- Click не дошёл до onConnectClick — нет записи в boot.log → crash до обработчика или процесс умер до логирования
- Топ-1 гипотеза: LaunchedEffect в setContent → exception → логируется в in-memory LogBuffer, **не в BootFileLogger** → crash теряется
- Альтернативная гипотеза (alt-vpn-dev): Hilt DI граф → UnsatisfiedLinkError при System.loadLibrary("hev-socks5-tunnel") → native crash или JVM exception до flush
- Hilt crash в Service — только вторая фаза (после click → startService → onCreate), на текущий silent crash не работает

**Decisions Made:**
- POST_NOTIFICATIONS — **не причина** текущего crash (startForeground не достигнут), alt-vpn-dev отозвал "CRITICAL" статус
- registerForActivityResult timing — **норма**, field initializer до super.onCreate()
- SecurityWatchdog — **безопасен**, graceful return с Toast
- Android 15 specialUse FGS — **не breaking change** для VPN

**Lessons Learned:**
- Exception в Compose LaunchedEffect логируется в in-memory буфер (AppLogger/LogBuffer), не в BootFileLogger → crash невидим в boot.log
- Hilt @Inject в VpnService: если провайдер throws (напр. .so загрузка), Hilt граф ломается — exception не пойман в startCommand try-catch
- Nubia ROM может быть stricter с permission enforcement — нужно тестирование на реальном устройстве

**Action Items:**
- Запросить у пользователя: полный boot.log после reboot, filesDir/crashes/, adb logcat -b crash, dumpExitReasons (task #22)
- Исследовать Android 15 Compose Activity silent crash patterns (task #21)

- Вторичный fix: collectAsState → collectAsStateWithLifecycle в MainScreen
- Решить подход к POST_NOTIFICATIONS permission check (не критично сейчас, но нужно для Android 13+)
