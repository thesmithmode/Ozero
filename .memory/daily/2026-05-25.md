### Session: v0.2.1 install failure — root cause + fix

**Цель:** Найти причину "пакет недействителен" для v0.2.1 APK, починить, выпустить заново.

**Ложные гипотезы (отвергнуты):**
- V1 signing disabled → включили V1=true, ошибка осталась. apksigner verify показал V1=false для обоих APK — V1 signing был red herring
- R8 stripped manifest class → все classes в DEX (assertions добавлены, прошли). Не root cause
- INDEX.LIST от Objenesis/Kryo в APK → подозрительный артефакт, но не install blocker

**ROOT CAUSE: versionCode downgrade!**
- v0.2.0 APK: versionCode=1155, versionName="0.2.0"
- v0.2.1 APK: versionCode=618, versionName="0.1.11"
- Android отклоняет install с LOWER versionCode как "пакет недействителен"
- Причина регрессии: git history rewrite (удаление AI authorship via rebase+amend+force-push) уменьшило `git rev-list --count HEAD` с ~1155 до ~618
- versionName тоже регрессировал: `git describe --tags` подхватывает старый v0.1.11 тег (v0.2.0 тег не reachable после history rewrite)

**Фикс (9866f5db):**
- versionCode = max(commitCount + 1500, 2000) — offset + floor, безопасно выше v0.2.0 (1155)
- versionName = env OZERO_VERSION_NAME (из release.yml tag resolver), fallback git describe
- CI assertion: versionCode >= 2000 вместо > 1

**Также применены (588046f9, ранее):**
- enableV1Signing = true (не root cause, но совместимость)
- DEX assertions для всех manifest-классов (не root cause, но важный guard)

## Concepts

### versionCode regression after git history rewrite

`git rev-list --count HEAD` зависит от количества коммитов. History rewrite (rebase, amend, filter-branch, force-push) может УМЕНЬШИТЬ commit count. Это ломает versionCode монотонность → Android отклоняет install как downgrade.

Защита: versionCode = max(commitCount + OFFSET, FLOOR). FLOOR = последний известный versionCode + запас. CI assertion с жёстким порогом.
