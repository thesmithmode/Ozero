### Session (22:08)

**Context:** Диагностика неработающих WARP и URnetwork движков в Ozero

**Key Exchanges:**
- WARP: `awgTurnOn` JNI fail → `libam-go.so` не загружена. Причина: вызываем `GoBackend.awgTurnOn` напрямую без инстанцирования `AbstractBackend`, где `ReLinker.loadLibrary(context, "am-go")` в конструкторе
- URnetwork: после снятия Nubia-guard (commit 47d0156) → SIGABRT при `auto-creating guest network`. Корень: env=`prod` вместо `main` + не выставляем bundle поля (linkHostName, migrationHostName, wallet) через `updateNetworkSpace`

**Decisions Made:**
- WARP fix: lazy `loadOnce("am-go")` в `RealWarpSdkBridge` через `System.loadLibrary` (без ReLinker, т.к. universal APK без splits)
- URnetwork fix: сменить env на `main` + выставить bundle поля перед `networkCreate`

**Lessons Learned:**
- amneziawg-android AAR грузит `libam-go.so` через ReLinker в `AbstractBackend` ctor — если вызывать JNI-методы напрямую минуя ctor, нативка не загружена
- URnetwork Go SDK требует корректного networkSpace (env=`main` + host/wallet поля), иначе SIGABRT при guest network creation
- Референсные проекты в `.claude/Контекст/` — ключевой источник правильных параметров (env, bundle fields)

**Action Items:**
- Завершить URnetwork fix: env `prod`→`main`, добавить `updateNetworkSpace` с bundle полями из референса
- Проверить оба движка после фиксов через CI + runtime лог
