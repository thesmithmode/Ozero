**Context:** Добор branch coverage в `common-vpn` до gate 95% через тесты и узкий refactor `StartSequenceCoordinator`.

**Key Exchanges:**
- Исправлен compile error в новом тесте: `healthMonitor.start` является `suspend`, поэтому для MockK нужен `coVerify`, а не `verify`.
- Добавлялись крупные пачки тестов без запуска на каждый кейс: `EngineWatchdogCoordinator`, `OzeroVpnServiceActionDispatcher`, `TunnelController`, `TunInterfaceStats`, `StartSequenceCoordinator`, `OzeroVpnService`, `ShutdownCoordinator`.

- Свежий XML показал, что часть остатка находится в Android/service/native wrapper branches и Kotlin/safe-call synthetic-like branches, а не в бизнес-логике.

**Decisions Made:**
- Не мокать final Android lock classes; покрывать lock-related ветки через реальные locks или избегать хрупких тестов.

**Lessons Learned:**
- `stopping=true` в custom TUN test нельзя выставлять до `run()`, иначе будет ранний return; флаг нужно включать внутри `establish()`, чтобы покрыть ветку после получения TUN.
- MockK chaining для throw-then-success лучше заменять явным счётчиком, чтобы тест не был хрупким.
- Safe-call defaults в горячей coordinator-логике создают много branch coverage шума; локальное вычисление effective settings сохраняет поведение и уменьшает искусственные ветки.

### Session (01:17)

**Context:** Добивали branch coverage модуля `common-vpn` до порога 95% без исключения сервиса из gate.

**Key Exchanges:**
- `NativeHevTunnelGateway` перевели с прямого `PersistentLoggers.instance?.*` на фасад `PersistentLoggers.*`, чтобы убрать ложные nullable branch misses и сохранить единый fallback в `Log.*`.
- Source-sentinel для pre-nativeStart был обновлён с ожидания `instance?.info` на фасадный `PersistentLoggers.info`.
- В `OzeroVpnService` добавили тестируемый helper для ownerUid guard и прямые unit tests рядом с существующими Robolectric/`NetworkCapabilities` тестами.
- Обнаружился compatibility нюанс: `NetworkCapabilities.ownerUid` может отсутствовать в Robolectric/runtime API surface и давать `NoSuchMethodError`.
- В `HevTunnelGateway` упростили stats-poller loop с `while (!isInterrupted && started)` до `while (started.get())`; shutdown semantics сохраняются через `interrupt()` и `InterruptedException`.
- В `StartSequenceCoordinator` убрали доказуемо лишние nullable-ветки `firstOrNull`, заменив на `first`, потому что selection step уже доказывает наличие plugin.
- Source-sentinel для coordinator был обновлён под `plugin.preflight()` вместо старой nullable-формы.

**Decisions Made:**
- OwnerUid helper сделан fail-closed: если API недоступен, сеть не считается собственной, что соответствует старому поведению на API без `ownerUid`.
- `runCatching` в ownerUid helper заменён на явный `try/catch`, потому что это проще для JaCoCo и ловит только нужную API-surface проблему.
- Source-contract для ownerUid guard перенесён на проверку helper-а/всего source, так как инвариант теперь живёт в `isOwnVpnNetwork`, а не напрямую в body `logActiveExternalVpn`.

**Lessons Learned:**
- Nullable-фасады и safe-call вокруг фактически non-null Android/service API могут создавать ложный branch debt в JaCoCo.
- Source-contract tests могут ломаться при структурно эквивалентных изменениях; их нужно привязывать к инварианту, а не к старой форме реализации.
- Robolectric/API surface может отличаться по наличию новых Android методов, даже если compile API их видит; такие обращения нужно guarded-тестировать.

**Action Items:**
- Дождаться финального `common-vpn` прогона и проверить coverage XML на достижение 95% branch threshold.

**Lessons Learned:**
- Source-contract body extraction может обрезать функцию раньше нужного helper call; проверка по всему source надежнее и не ослабляет инвариант.
