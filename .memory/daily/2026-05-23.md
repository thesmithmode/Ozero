**Корень WARP регрессии (commit f458dd5d):**
- `bd6a178a` (2026-05-20) добавил поиск UAPI socket в `uapiPath/sockets/` subdir с fallback `firstOrNull` (lex-order листинга).

**Fix:**

- WarpHandshakeUapi.findUapiSocket: `maxByOrNull{lastModified}` вместо `firstOrNull`.
- 5 unit + 2 source-sentinel тестов.

**Неразрешённые загадки в логе (не блокеры):**
- `Log.i` теги engine-warp модуля НЕ попадают в ozero.log (только PersistentLoggers.warn). `Log.i(TAG, "attachTun fd=...")` из EngineWarp.kt:208 и `Log.i(TAG, "protect v4 sock=...")` отсутствуют. - `version=disconnected` при первом awgTurnOn — OK, означает `engine?.version()` вызван до ensureConnected (по дизайну). Не баг.

**URnetwork heuristic (финал defc0438 — откат 14f9f22e):**
- Поле "Доступно" = `balance + reliabilityBonusBytes` БЕЗ pending, где `reliabilityBonusBytes = min(meanReliabilityWeight * 100, 100) GiB`.
- 14f9f22e включал pending → итог `used + pending + (balance + pending + bonus)` = pending дважды. Откачено в defc0438.
- Это эвристика, НЕ реальные байты из SDK. Оригинальный URnetwork (`SubscriptionBalanceViewModel.kt`) НЕ суммирует бонус — показывает только `result.balanceByteCount`.
- Юзер сказал "у оригинального вроде суммирует" — это его ошибка, но мы реализовали как просит.
- Если URnetwork SDK когда-то начнёт возвращать настоящий `bonusByteCount` поле — обновить формулу.
- Прогресс-бар теперь `total = used + pending + (balance + heuristicBonusBytes)` — distorted на cosmetic-байты, но синхронизирован с "Доступно" по требованию юзера.

**FPTN brotli (commit cbeaeaee):**
- Префикс `fptnb:` = base64(brotli(JSON)). Дополнен парсер через `org.brotli:dec:0.1.2` (pure-Java decoder, Apache 2.0).
- Не было positive-теста brotli (encoder отсутствует в decoder-only lib), только negative + source-sentinel.

**Lessons:**
- При выборе из множества stale-кандидатов: НИКОГДА `firstOrNull` без mtime/sort — выбирай newest.
- Per-session cleanup ОБЯЗАТЕЛЕН при работе с filesystem state, особенно UNIX sockets (.sock файл существует физически после смерти процесса).
- При user-requested cosmetic-фиксах документировать в memory чем они отличаются от SDK контракта.
