# Ozero — юридический контракт reverse-engineering

Этот документ описывает юридические рамки разработки Ozero и использование reverse-engineering техник при работе с proprietary приложениями.

---

## Используемые приложения-референсы

При разработке Ozero выполнен анализ архитектуры следующих приложений для изучения техник обхода блокировок:

### КИБЕРЩИТ-X (closed source)

- **Статус**: Proprietary приложение (не open-source)
- **Использование**: JADX-декомпиляция для анализа поведения и архитектуры
- **Какое содержимое НЕ копируется**: Байткод, ресурсы, строки, конфигурации, сетевые протоколы, криптографические ключи
- **Что разрешено**: Понимание общей архитектуры, логики выбора протоколов, стратегий обхода DPI
- **Бинарь в репо**: НЕ копируется
- **Лицензия**: Proprietary — используется только для анализа, не для интеграции или воспроизведения кода

### CYBERPORTAL_X (closed source)

- **Статус**: Proprietary приложение (не open-source)
- **Использование**: JADX-декомпиляция для анализа поведения и архитектуры
- **Какое содержимое НЕ копируется**: Байткод, ресурсы, строки, конфигурации, шифрование трафика, логика DNS-обхода
- **Что разрешено**: Понимание стратегий фрагментации пакетов, методов маскировки SSL/TLS-handshake
- **Бинарь в репо**: НЕ копируется
- **Лицензия**: Proprietary — используется только для анализа, не для интеграции или воспроизведения кода

### PortalConnect (closed source)

- **Статус**: Proprietary приложение (не open-source)
- **Использование**: JADX-декомпиляция для анализа поведения и архитектуры
- **Какое содержимое НЕ копируется**: Байткод, ресурсы, сетевой стек, алгоритмы обхода, proprietary транспорты
- **Что разрешено**: Понимание UI-паттернов управления конфигурациями, логики выбора серверов, механик переключения протоколов
- **Бинарь в репо**: НЕ копируется
- **Лицензия**: Proprietary — используется только для анализа, не для интеграции или воспроизведения кода

### ByeByeDPI (open source, GPLv3)

- **Статус**: Open source, свободно доступный исходный код
- **Использование**: Анализ исходников на GitHub, возможна прямая интеграция
- **Какое содержимое разрешено**: Полный исходный код, алгоритмы, реализация техник DPI-обхода
- **Интеграция**: ByeByeDPI распространяется под GPLv3, а Ozero — тоже под GPLv3. Copyleft-требование (распространение в том же формате, с доступом к исходникам) полностью удовлетворено
- **Бинарь в репо**: Не копируется целиком; код может быть адаптирован и интегрирован при соблюдении GPLv3
- **Лицензия**: GPLv3 — полная совместимость

### Invizible Pro (open source, GPLv3)

- **Статус**: Open source, частично с proprietary компонентами (DNSFilter backend), но основной код свободен
- **Использование**: Анализ исходников на GitHub, возможна интеграция open-source фрагментов
- **Какое содержимое разрешено**: Исходный код компонентов Tor, обфускации, DNS-фильтрации (кроме proprietary backend)
- **Интеграция**: Open-source части Invizible Pro (например, обёртка вокруг Tor) распространяются под GPLv3. Интеграция этих фрагментов в Ozero возможна при сохранении атрибуции и лицензии
- **Бинарь в репо**: Не копируется; open-source код адаптируется с сохранением авторства
- **Лицензия**: GPLv3 (open-source части) — совместима

## Upstream-сборка Xray

**Xray-core** — основной proxy-engine Ozero — собирается **самостоятельно** из публичного исходного кода.

### Процесс сборки

1. Исходники берутся напрямую с GitHub: https://github.com/XTLS/Xray-core
2. Сборка выполняется через `gomobile bind` в native Android AAR-артефакт
3. AAR **не берётся** из готовых сторонних сборок или бинарных репозиториев
4. Исходники компилируются в чистом окружении (Docker/CI) с контролем версии Go и зависимостей

### Обоснование

- **Воспроизводимость**: Сборка можно повторить на другой машине с тем же кодом и тем же результатом
- **Trust chain**: Нет зависимости от сторонних собирателей, которые могут вставить вредоносный код
- **F-Droid требования**: Для включения в F-Droid репозиторий все зависимости должны быть воспроизводимо собираемы из исходников. Proprietary AAR-артефакты не допускаются
- **Переносимость**: В случае форков Xray или появления уязвимостей можно быстро пересобрать с другой версией

## Совместимость лицензий

### Основная лицензия Ozero

Ozero распространяется под **GNU General Public License v3 (GPLv3)**. Это сильная copyleft-лицензия, которая требует:
- Распространение исходников вместе с приложением
- Сохранение права пользователей модифицировать приложение
- Распространение модификаций под той же лицензией

### Совместимость с используемыми компонентами

| Лицензия компонента | Совместимость с GPLv3 | Статус |
|---------------------|----------------------|--------|
| MPL-2.0 (Xray-core) | ✅ Совместима | Разрешено использовать |
| MIT (AmneziaWG основной, hev-socks5-tunnel, BouncyCastle) | ✅ Совместима | Разрешено использовать |
| GPLv2 (AmneziaWG фрагменты) | ⚠️ Требует уточнения | Потенциальная несовместимость (см. ниже) |
| BSD-2-Clause (obfs4proxy) | ✅ Совместима | Разрешено использовать |
| BSD-3-Clause (Tor, Snowflake, NaiveProxy) | ✅ Совместима | Разрешено использовать |
| Apache-2.0 (OkHttp, Ktor Client, Kotlin, AndroidX) | ✅ Совместима | Разрешено использовать |
| GPLv3 (ByeDPI) | ✅ Совместима | Полная совместимость, код может быть встроен |

### ⚠️ Потенциальная проблема: GPLv2 в AmneziaWG

AmneziaWG-android содержит фрагменты, распространяемые под **GPLv2** (исходный WireGuard). Проблема в том, что GPLv2 не совместима с GPLv3 при прямой интеграции кода.

**Решение**:
- Если интегрируется только MIT-части AmneziaWG → проблемы нет
- Если интегрируются GPLv2-части → требуется либо:
  1. Согласие авторов AmneziaWG на распространение под GPLv3, либо
  2. Связь через SOCKS5/IPC (не копирование кода) вместо прямой встройки, либо
  3. Лицензирование Ozero под GPLv2-3 compatible (например, `GPLv3 or later at user's choice`)

**Текущий статус**: Требует уточнения при интеграции AmneziaWG. На данный момент предполагается минимальная встройка или использование через отдельный процесс.

## F-Droid requirements

Для включения в официальный репозиторий F-Droid Ozero должен соответствовать следующим требованиям:

### Отсутствие отслеживания

- ✅ Нет встроенных систем аналитики, отслеживания, реклам
- ✅ Нет сбора данных о поведении пользователя
- ✅ Нет телеметрии (кроме явных уведомлений об ошибках на усмотрение пользователя)

### Отсутствие proprietary зависимостей

- ✅ Все основные зависимости — open source
- ✅ Google Play Services не используются для основного функционала
- ✅ Proprietary binary зависимостей нет

### Воспроизводимая сборка

- ✅ Все исходники находятся в репозитории (или указаны явные версии от upstream)
- ✅ Сборка выполняется из исходников без proprietary артефактов
- ✅ Зависимости versioned и reproducible (go.mod, gradle версии, pin версий Android SDK)
- ✅ Возможен полный rebuild в контролируемом окружении

### Автоматизированная build-процедура

- ✅ fdroid-compatible build.gradle с правильной target SDK
- ✅ Отсутствие ручных шагов сборки
- ✅ Поддержка `gradle assembleRelease` или `gradle bundleRelease`
- ✅ Версионирование через semantic versioning (MAJOR.MINOR.PATCH)

## Export compliance

Ozero содержит криптографические функции (TLS, Ed25519, AES), которые в некоторых юрисдикциях рассматриваются как криптография и подпадают под export control.

### F-Droid стандарт

Для F-Droid достаточно добавить стандартную декларацию в metadata:

```
ExoPlayer: No
NonFreeNet: No
NonFreeAdd: No
NonFreeDep: No
DisabledForSandbox: No
Disabled: No

Categories: Internet
License: GPLv3
```

В приложении явно указать:
- Приложение использует криптографию стандартных алгоритмов (TLS, Ed25519)
- Криптография встроена в Android Framework и не добавляет extra restrictions
- Для США: Исключение на криптографию в open-source приложениях (EAR Section 740.13(e))

### Дополнительные юрисдикции

- **Россия**: Требует регистрации средств криптографии при массовом распространении. Open-source приложения в App Store получают исключение как «не подпадающие под ограничения».
- **Европа**: GDPR требует уведомления о сборе данных (Ozero не собирает данные → нет требований).
- **Китай**: VPN-приложения ограничены на официальных маркетах; распространение через GitHub/F-Droid остаётся возможным.

Ozero распространяется через GitHub и потенциально через F-Droid. Во всех этих каналах действуют стандартные export rules для open-source криптографии, которые считаются освобождены.
